# Проект колеровки и расчёта материалов

Консольный Node.js-проект для расчёта декоративных покрытий, материалов и колеровки.  
Главная точка входа — **`START.desktop`**: он запускает приложение в 2 клика, после чего внутри выбирается один из трёх основных режимов работы с базами и системой расчёта.

## Быстрый старт

### Запуск в 2 клика

Используй файл:

```bash
START.desktop
```

Он нужен как основной лаунчер проекта:
- запускает приложение без ручного ввода длинных команд;
- открывает главное меню;
- даёт выбрать один из **трёх основных режимов работы**.

Если `.desktop` не запускается автоматически, проверь права на запуск файла и настройки среды рабочего стола.

### Ручной запуск

Если нужно запускать напрямую по файлам:

```bash
node mainmenu.js
```

```bash
node made-colerovka.js
```

## Основные режимы

После запуска через `START.desktop` пользователь попадает в главное меню и выбирает нужный сценарий работы.

### 1. Калькулятор расчёта

Основной пользовательский режим:
- выбор системы;
- ответы на вопросы;
- подбор материалов;
- расчёт фасовок и стоимости;
- расчёт колеровки;
- вывод общего итога.

Точка входа вручную:

```bash
node mainmenu.js
```

### 2. Админка систем

Режим настройки систем колеровки:
- создание и редактирование систем;
- настройка вопросов;
- настройка слоёв;
- настройка цветов и коэффициентов;
- настройка типов колеровки;
- сохранение изменений.

Точка входа вручную:

```bash
node made-colerovka.js
```

### 3. Работа с базами материалов

Третий режим используется для работы с базами и связанными данными проекта — в зависимости от текущей сборки это могут быть отдельные служебные CLI-файлы для редактирования, проверки или подготовки базы материалов.

## Что умеет проект

- Выбор системы отделки.
- Опрос пользователя по вопросам системы.
- Расчёт материалов по слоям и фасовкам.
- Выбор цвета.
- Расчёт колеровки по формуле.
- Добавление стоимости колеровки в итог по слою и в общий итог.

## Основные файлы

| Файл | Назначение |
|------|------------|
| `START.desktop` | Основной запуск проекта в 2 клика. |
| `mainmenu.js` | CLI-калькулятор расчёта материалов и колеровки. |
| `made-colerovka.js` | Админка систем колеровки. |
| `system-color-base.js` | Реестр систем: имена, активность, пути к system-файлам. |
| `materials-db.js` | Общая база материалов. |
| `type-colerovka-math.js` | Типы колеровки и формулы расчёта. |

## Данные проекта

### Система
Система описывает:
- `name` — имя;
- `description` — описание;
- `questions` — вопросы пользователю;
- `layers` — слои;
- `colorConfig` — настройки цветов и модели колеровки.

### Цвет
Цвет обычно хранится в отдельном реестре и содержит:
- `id`;
- `name`;
- `shortName`;
- `value`;
- `coloringTypeId`;
- `application` — где и как цвет участвует в колеровке.

### Тип колеровки
Тип колеровки хранит:
- товар колеровки;
- цену;
- размер тубы;
- правило округления;
- формулу расчёта.

Базовая формула:

```txt
ceil((sum(layerBucketCount * applicationCoefficient) * colorValue) / tubeSize)
```

## Как работает расчёт

### Материалы
Для каждого слоя калькулятор:
1. определяет материал;
2. ищет его в `materials-db.js`;
3. подбирает фасовки;
4. раскладывает площадь по фасовкам;
5. считает стоимость;
6. выводит итог по слою.

### Колеровка
Для модели `value-linked`:
1. берётся выбранный цвет;
2. находится `coloringTypeId`;
3. загружается тип колеровки из `type-colerovka-math.js`;
4. берётся `application` по слоям и фасовкам;
5. считается количество туб;
6. стоимость добавляется в слой и общий итог.

## Важные правила

### 1. Названия материалов должны совпадать
Если слой ссылается на материал `Грунт ФОН`, то в `materials-db.js` он должен называться точно так же.

### 2. Названия фасовок должны совпадать
Если в `application` указана фасовка `2,5л/2,5кг`, то в базе материалов она должна называться так же.

### 3. После изменений в админке нужно сохранять
После правок вопросов, слоёв, цветов и коэффициентов обязательно нажимай `Сохранить`.

## Как добавить новую систему

Через админку:

```bash
node made-colerovka.js
```

Обычный порядок:
1. Создать систему.
2. Заполнить карточку.
3. Добавить вопросы.
4. Добавить слои.
5. Настроить цвета.
6. Настроить типы колеровки.
7. Сохранить.

## Как добавить новый материал

Есть два варианта:
- добавить материал в профильный файл категории;
- проверить, что он попал в `materials-db.js`, потому что именно его использует `mainmenu.js`.

## Быстрая проверка после изменений

Проверь:
- запускается ли `START.desktop`;
- открывается ли главное меню;
- запускается ли `node made-colerovka.js`;
- сохраняются ли изменения после перезапуска;
- запускается ли `node mainmenu.js`;
- находится ли материал без ошибки `Не найден материал`;
- считается ли колеровка;
- попадает ли она в итог по слою и в общий итог.
