# Проект колеровки и расчёта материалов

Этот проект — консольная система для расчёта декоративных покрытий, материалов и колеровки. В нём есть две основные части: пользовательский калькулятор расчёта и админка для настройки систем, вопросов, слоёв, цветов и логики колеровки.

## Что делает проект

Проект позволяет:
- выбирать систему отделки;
- отвечать на вопросы по системе;
- рассчитывать нужные материалы по слоям и фасовкам;
- выбирать цвет;
- рассчитывать колеровку по формуле;
- добавлять стоимость колеровки к нужному слою и в общий итог.

## Основные сценарии

### 1. Пользовательский расчёт

Главный вход для расчёта:

```bash
node mainmenu.js
```

Этот файл:
- показывает список систем;
- задаёт вопросы пользователю;
- подбирает материалы;
- считает фасовки и стоимость;
- считает колеровку;
- печатает итог по слоям и общий итог.

### 2. Админка систем

Главный вход для настройки:

```bash
node made-colerovka.js
```

Этот файл нужен для:
- создания новых систем;
- редактирования существующих систем;
- настройки вопросов;
- настройки слоёв;
- настройки цветов и коэффициентов;
- настройки типов колеровки;
- сохранения изменений в файлы проекта.

## Как связаны основные сущности

### 1. Система

Система описывается в файле вроде `traverto-system-colorr.js`.

Внутри системы обычно есть:
- `name` — имя системы;
- `description` — описание;
- `questions` — список вопросов пользователю;
- `layers` — список слоёв;
- `colorConfig` — настройки реестра цветов и модели колеровки.

### 2. Цвет

Цвет лежит в отдельном файле, например `traverto-naturale-color.js`.

У цвета есть:
- `id`;
- `name`;
- `shortName`;
- `value` — числовое значение цвета;
- `coloringTypeId` — ссылка на тип колеровки;
- `application` — где и как этот цвет участвует в колеровке.

### 3. Тип колеровки

Тип колеровки лежит в `type-colerovka-math.js`.

Пример логики:
- есть товар колеровки, например `acs metrico 80`;
- есть цена одной тубы;
- есть размер тубы;
- есть правило округления;
- есть формула расчёта количества.

Текущая формула в проекте:

```txt
ceil((sum(layerBucketCount * applicationCoefficient) * colorValue) / tubeSize)
```

Это означает:
1. Для нужного слоя берётся количество упаковок.
2. Количество умножается на коэффициент применения.
3. Все значения суммируются.
4. Сумма умножается на значение цвета.
5. Делится на размер тубы.
6. Результат округляется вверх до целого.

### 4. Материалы

Материалы лежат в общей БД `materials-db.js`.

Структура такая:

```js
{
  "Россия": {
    "Грунт ФИКС СУПЕР": [
      {
        variant: "1л/1кг",
        coverage_m2: 20,
        price_rub: 1650
      }
    ]
  }
}
```

То есть:
- сначала идёт страна;
- внутри — название материала;
- у материала массив фасовок;
- у фасовки есть вариант, укрывистость и цена.

## Логика расчёта

### Расчёт материалов

`mainmenu.js` для каждого слоя:
1. определяет нужный материал;
2. ищет его в `materials-db.js`;
3. берёт доступные фасовки;
4. раскладывает нужную площадь по фасовкам;
5. считает стоимость по каждой фасовке;
6. выводит итог по слою.

### Расчёт колеровки

Если система использует `value-linked`, тогда:
1. берётся выбранный цвет;
2. у цвета читается `coloringTypeId`;
3. по нему находится формула в `type-colerovka-math.js`;
4. из `application` определяется, какие слои и фасовки реально колеруются;
5. считается количество туб колеровки;
6. количество округляется вверх;
7. стоимость колеровки выводится рядом со слоем;
8. стоимость добавляется в общий итог.

## Как добавить новую систему

Через админку:

```bash
node made-colerovka.js
```

Дальше обычно порядок такой:
1. Создать новую систему.
2. Заполнить карточку системы.
3. Добавить вопросы.
4. Добавить слои.
5. Настроить цвета.
6. Настроить типы колеровки.
7. Сохранить.

## Как добавить новый материал

Есть два пути:

### Вариант 1. Добавить в профильный файл
Например:
- грунты глубокого проникновения — в `ggp.js`;
- грунты-подложки — в `grunt-podlozhka.js`;
- декоративные штукатурки — в `dekor-shtukaturki.js`;
- лаки — в `finish-laki.js`.

### Вариант 2. Проверить объединение
После добавления материала проверь, что он попадает в `materials-db.js`, потому что именно этот файл импортирует `mainmenu.js`.

## Что важно помнить

### 1. Названия материалов должны совпадать

Если в системе слой называется:

```txt
Грунт ФОН
```

то в `materials-db.js` материал должен называться точно так же:

```txt
"Грунт ФОН"
```

Иначе `mainmenu.js` не найдёт материал.

### 2. Названия фасовок тоже должны совпадать

Если у цвета в `application` указано:

```txt
"2,5л/2,5кг"
```

то точно такое же имя фасовки должно быть у материала в БД.

### 3. После изменений в админке нужно сохранять

В `made-colerovka.js` теперь есть ручное сохранение. После изменения вопросов, вариантов, слоёв или цветов надо нажимать `Сохранить`, чтобы данные записались в файлы.

## Быстрый чек после изменений

После правок стоит проверять:
- запускается ли `node made-colerovka.js`;
- сохраняются ли изменения после перезапуска;
- запускается ли `node mainmenu.js`;
- находится ли материал без ошибок `Не найден материал`;
- считается ли колеровка;
- попадает ли колеровка в итог по слою и общий итог.

## Пример рабочего сценария

1. Открыть админку:

```bash
node made-colerovka.js
```

2. Настроить систему и сохранить.

3. Открыть калькулятор:

```bash
node mainmenu.js
```

4. Выбрать систему, цвет, метраж, лак.

5. Получить:
- фасовки по каждому слою;
- стоимость материалов;
- количество колеровки в штуках;
- стоимость колеровки;
- общий итог.
